1 Heuristic paper

"A Formal Basis for the Heuristic Determination of Minimum Cost Paths", Peter E. Hart, Nils J. Nilsson, and Bertram Raphael.

2 Definitions

For any subgraph \(G_s\) and any goal set \(T\) and starting state \(s\):

Let \(f^*(n)\) be the actual cost of an optimal path constrained through \(n\), from \(s\) to \(n\) to a preferred goal node \(n\).

From this definition, the ('unconstrained') optimal path can be written as \(f^*(s) = h^*(s)\), where \(h^*(n)\) is the optimal path from node \(n\) to the preferred goal node of \(s\).

If \(n\) is in the optimal path, then \(f^*(n) = f^*(s)\).

In particular, \(f^*(n') > f^*(s)\) for every node \(n'\) not on the optimal path.

In addition we will also define a \(g^*(n)\) as the optimal cost path from \(s\) to \(n\). Thus \(f^*(n)\) can be divided into two terms:

\(f^*(n) = g^*(n) + h^*(n)\)

3 Heuristics and best cost so far

A heuristic algorithm can estimate all of these cost, let \(f,g,h\) be estimates for the corresponding optimal costs \(f^*,g^*,h^*\). In particular,

\(g(n) \ge g^*(n)\)

As we are exploring the graph \(g(n)\) is our estimate for the cost from \(s\) to \(n\). So think of \(g(n)\) as the "best cost so far" to node \(n\).

\(h(n)\) is the estimated cost from \(n\) to a preferred target node. It is an estimate from some heuristic that we apply to the problem. For example one might estimate the trip distance between SF and NY to be the straight line distance on a map, whereas the optimal distance via driving by car will most likely be longer.

\(f(n)\) is the estimate for the whole constrained path from \(s\) to a preferred target node from \(n\) has too estimates:

\(f(n) = g(n) + h(n)\)

The estimate from the best cost so far to node \(n\) (from the normal Dijkstra like updates via frontier nodes) and the heuristic cost for the rest of the trip from that node \(n\) to its preferred target node.

I have been referring to the preferred target node because the goal set \(T\) could be composed of various desirable end states, for example winning positions in a chess game.

4 Graph search algorithm

Some terminology for graph algorithm, typically we select a node from the frontier nodes which has the best cost estimate so far. This node is then expanded, i.e. we look at the possible actions from this node, and update or generate new nodes depending on the whether this node touches other nodes in the frontier or adds new nodes to the frontier. The expanded node then becomes part of the explored set (or closed set) and any nodes that were created become part of the frontier set.

This graph has a starting node \(s\).

Starting with \(s\) there are two neighbors \(n_1\) and \(n_2\).

A-star "expands" from the \(s\) and generates estimates for \(g(n_1)\) and \(g(n_2)\) to \(3\) and \(7\).

Suppose \(n_1\) is expanded with neighbors \(n_2\) and \(n_3\), at this stage we can update the cost to \(g(n_2)\) to 5 and generate an estimate \(g(n_3) = 5\).

To generate a estimate for \(h(n)\) we need some heuristic for the cost to the target.

5 Admissible

From the above it is important to note that \(g\) is the best estimate thus far.

There is a lemma saying if there is a unexplored node \(n\) in some optimal path \(P\), then there is an a node \(n'\) on \(P\) with \(g(n') = g*(n')\). We will not prove this lemma, but basically it says there is some node in the frontier that will be part of the optimal path. The AIMA book calls this the graph separation property. Basically the GRAPH-SEARCH algorithm creates a fronties which separates state-space graph into two regions an explored and an unexplored region, so every path from some initial state \(s\) has to pass through this frontier to a goal state, thus there must be a node \(n'\) in the frontier which is part of the optimal path. We will just call this the GRAPH-SEARCH separation property.

If we now assume that

\(h(n') \le h*(n')\) (Admimissible)

then we can show that

\(f(n') \le f*(s)\)

The proof goes as follows, say \(g(n') = g*(n')\), then

\begin{eqnarray} f(n') &=& g*(n') + h(n') \\ &\le& g*(n') + h*(n') \\ &=& f(n') = f(s) \end{eqnarray}
